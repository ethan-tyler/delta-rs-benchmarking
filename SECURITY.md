# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| `main` | Yes |
| older branches and ad hoc snapshots | No |

Security fixes land on `main` first. If a tagged release process is introduced later, the latest maintained release should be treated as the only supported release line unless this document says otherwise.

## Reporting a Vulnerability

Do not open a public issue for suspected security vulnerabilities.

Use one of these paths instead:

- GitHub Security Advisories for private disclosure when available on the repository.
- Email `ethan@urbanskitech.com` with the subject line `delta-rs-benchmarking security report`.

Include:

- affected commit, branch, or workflow
- reproduction steps or proof of concept
- impact assessment
- any suggested mitigation or patch direction

You should receive an acknowledgment within 5 business days. If the issue is confirmed, the next response should include a remediation plan or a request for more information.
